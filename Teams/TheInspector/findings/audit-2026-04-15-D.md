## Findings

### QO-001: Route handlers bypass service layer — direct store calls
- **Severity:** P2
- **Category:** architecture-violation
- **Files:**
  - `Source/Backend/src/routes/workItems.ts` — 8 direct `store.*` calls
  - `Source/Backend/src/routes/workflow.ts` — 9 direct `store.*` calls
  - `Source/Backend/src/routes/intake.ts` — 2 direct `store.createWorkItem` calls
- **Detail:** CLAUDE.md architecture rule: *"No direct DB calls from route handlers — use the service layer."* All three route files call `store.createWorkItem`, `store.findById`, `store.findAll`, `store.updateWorkItem`, and `store.softDelete` directly. The store is the persistence layer; routes should delegate to a service (e.g. `workItemService`) that owns this logic.
- **Recommendation:** Extract a `workItemService.ts` with functions like `createItem`, `getItem`, `listItems`, `updateItem`, `deleteItem`. Route handlers call service functions only; the store remains an implementation detail of the service. Workflow actions (`route`, `assess`, `dispatch`) already partially use services (`router`, `assessment`) — extend this pattern to CRUD.
- **Cross-ref:** TheFixer for implementation.

### QO-004: OpenTelemetry tracing not implemented
- **Severity:** P2
- **Category:** architecture-violation
- **Files:** `Source/Backend/src/utils/logger.ts`, `Source/Backend/package.json`
- **Detail:** CLAUDE.md observability rules and FR-021 require: "Use OpenTelemetry for distributed tracing", "Auto-instrument HTTP routes and database calls", "Propagate W3C `traceparent` header across service boundaries", "Trace/span IDs appear in logs." No `@opentelemetry/*` packages are in `package.json` and no span instrumentation exists in any source file. Logging and Prometheus metrics are implemented, but the tracing pillar is missing entirely.
- **Recommendation:** Add `@opentelemetry/sdk-node`, `@opentelemetry/auto-instrumentations-node`, configure a trace exporter. Add custom spans in `assessment.ts` and `router.ts` (critical paths). Inject `traceparent` response header in middleware. Link span IDs into log entries.
- **Cross-ref:** TheFixer (backend-coder scope).

### QO-006: Logger ignores NODE_ENV — no dev pretty-printing
- **Severity:** P3
- **Category:** pattern-violation
- **File:** `Source/Backend/src/utils/logger.ts:17-25`
- **Detail:** CLAUDE.md and FR-003 require "structured JSON logging in production, pretty-printing in development." The `emit()` function always calls `JSON.stringify()` regardless of `process.env.NODE_ENV`. Developers running locally see minified JSON instead of readable log output.
- **Recommendation:** Branch on `NODE_ENV`:
  ```ts
  if (process.env.NODE_ENV === 'production') {
    process.stdout.write(JSON.stringify(entry) + '\n');
  } else {
    console.log(`[${entry.level.toUpperCase()}] ${entry.message}`, entry.context ?? '');
  }
  ```
  Add a test verifying the NODE_ENV branch.
- **Cross-ref:** TheFixer.

- **Severity:** P3

- **Detail:** CLAUDE.md states disabled linting rules are a code pattern to flag. Both suppressions silence `react-hooks/exhaustive-deps` — a rule that guards against stale closure bugs in hooks. Neither suppression has a comment explaining *why* the dependency should be excluded.
- **Recommendation:** For each suppression, either (a) fix the hook to include all dependencies correctly, or (b) add a justification comment explaining why the dep is intentionally omitted (e.g. `// intentionally exclude `foo` — it is a stable ref that never changes`).

### QO-010: Two logger abstractions — `src/logger.ts` wraps `src/utils/logger.ts`
- **Severity:** P4
- **Category:** pattern-violation
- **Files:** `Source/Backend/src/logger.ts`, `Source/Backend/src/utils/logger.ts`
- **Detail:** `src/logger.ts` exists solely to provide a default export that normalizes string vs object log calls for routes that were written before `utils/logger.ts` settled on its API. This creates two logger files where one would suffice, and is documented in the wrapper itself as a compatibility shim.
- **Recommendation:** Consolidate: extend `utils/logger.ts` to accept `string | Record<string,unknown>` directly and export as default. Remove the wrapper once all callers are updated.
