3. Executive Summary

The duplicate/deprecated status feature is functionally complete — all 13 spec requirements are implemented. However, the audit reveals 4 critical (P1) findings that prevent an A/B/C grade:

  1. No authentication on any API endpoint (RED-001) — any anonymous caller can mark items as duplicate/deprecated, delete records, or approve features. This is the single highest-impact finding and pre-dates the current feature.
  2. N+1 query pattern on list endpoints (PERF-001) — the new duplicated_by computation fires a SQL query per row, adding to 3 existing per-row queries. A 100-item list triggers 401 SQL statements.
  3. Missing database indexes (PERF-002) — no index on status, duplicate_of, or other frequently filtered columns. Every list query is a full table scan.
  4. Handlebars critical CVE (DEP-001) — transitive dependency with CVSS 9.8 JavaScript injection vulnerability.

Operator action required: Items 1-3 should block production deployment. Item 4 is a transitive dep in the test toolchain, lower urgency but still critical by CVE score.

9. P1 Findings

P1 NEW RED-001: No Authentication on Any API Endpoint

File: portal/Backend/src/index.ts:26-67

Category: auth-bypass

Exploit scenario:

  1. Attacker sends PATCH /api/bugs/BUG-0001 {"status": "deprecated"} with no credentials
  2. No auth middleware exists — request flows directly to route handler
  3. Handler calls service, status is changed. Any item can be hidden by any anonymous caller.

Impact: Full unauthenticated write access to all mutating endpoints. Any caller can hide items, delete records, approve features.

Recommendation: Register JWT/session auth middleware before all API routes in index.ts. All mutating endpoints (PATCH, POST, DELETE) must require authentication.

P1 NEW PERF-001: N+1 Queries on List Endpoints (4 extra per row)

Files: bugService.ts:38-66, featureRequestService.ts:109-139

Category: n-plus-1

Detail: mapBugRow() and mapFRRow() fire 4 SQL queries per row: duplicated_by lookup, getBlockedBy, getBlocks, hasUnresolvedBlockers. 100 items = 401 SQL statements. The duplicated_by sub-query was introduced by this feature.

Recommendation: Replace per-row queries with batch queries using IN (...) clauses. Reduces from O(4N) to O(3) total SQL statements.

P1 NEW PERF-002: Missing Database Indexes on Status, duplicate_of, and Filter Columns

File: portal/Backend/src/database/schema.ts

Category: missing-index

Detail: No index on status (used in every list WHERE), duplicate_of (queried per row for duplicated_by), source, severity, or votes.feature_request_id. Every filter and sub-query triggers a full table scan.

Recommendation: Add idempotent CREATE INDEX IF NOT EXISTS for: idx_feature_requests_status, idx_bugs_status, idx_feature_requests_duplicate_of, idx_bugs_duplicate_of, idx_votes_fr_id, idx_feature_requests_source, idx_bugs_severity.

P1 NEW DEP-001: Handlebars Critical CVE (CVSS 9.8)

Package: handlebars@4.7.8 (transitive via jest/babel)

File: Source/Backend/package-lock.json

Category: CVE

Detail: 8 vulnerabilities including GHSA-2w6w-674q-4c4q (CVSS 9.8, JavaScript Injection via AST Type Confusion). Exploitable if handlebars precompiler is used in build pipeline.

Recommendation: Upgrade handlebars to >=4.7.9. Check transitive dependency path through jest/babel.
